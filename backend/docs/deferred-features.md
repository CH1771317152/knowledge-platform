# Deferred Features

本文档记录已经讨论过、设计上预留、但当前阶段明确暂不实现的能力。每个条目包含归属模块、暂缓原因和未来触发条件，避免后续遗忘。

## Content / Storage

### 分片上传与断点续传

归属模块：`storage`

当前决策：第一版只实现普通 PUT 预签名上传。

暂缓原因：S3/OSS 分片上传需要额外的 `init`、`part presign`、`complete`、`abort` 接口和上传会话状态，不适合作为 content 主流程第一版的阻塞项。

未来触发条件：

- 正文或附件体积明显增大。
- 普通 PUT 上传在网络波动下体验不可接受。
- 前端编辑器需要稳定支持大文件续传。

### 正文详情 GET 预签名 URL / CDN 直出

归属模块：`content`、`storage`

当前决策：第一版详情页由后端读取 OSS Markdown 正文并返回。

暂缓原因：后端代理读取权限控制最简单，适合第一版验证业务闭环。

未来触发条件：

- 详情页正文读取流量明显增大。
- 后端正文代理成为带宽瓶颈。
- 引入 CDN 或对象存储直出策略。

### 发布后的缓存刷新

归属模块：`content`、`cache`

当前决策：正式发布接口预留事件点，第一版不实现缓存刷新。

暂缓原因：平台级缓存策略尚未设计，当前 cache 模块只管理 Caffeine 本地缓存。

未来触发条件：

- 公开文章列表、详情页或作者主页引入缓存。
- 发布、取消发布、删除后需要主动失效缓存。

### 异步 Feed 流索引

归属模块：`content`、未来 Feed 模块

当前决策：正式发布接口预留 `PostPublishedEvent` 或 outbox 事件点，第一版不更新 Feed。

暂缓原因：Feed 模块尚未设计。

未来触发条件：

- 实现关注流、推荐流或首页信息流。
- 内容发布需要异步分发给关注者或推荐索引。

### 搜索索引 / 全文检索

归属模块：`content`、未来 Search 模块

当前决策：正文 Markdown 存 OSS，MySQL 不直接承担全文检索。

暂缓原因：全文检索需要独立搜索引擎或异步索引流程，第一版先完成发布闭环。

未来触发条件：

- 需要按正文内容搜索。
- 标签和标题检索无法满足产品需求。
- 引入 Elasticsearch、OpenSearch 或其他搜索服务。

### 正文版本历史与回滚

归属模块：`content`

当前决策：第一版保留 `body_version` 字段，但不实现历史版本列表、diff 和回滚。

暂缓原因：第一版只需要一个当前正文版本支撑发布流程。

未来触发条件：

- 用户需要查看历史版本。
- 需要撤销编辑或回滚发布内容。
- 审核或合规要求保留正文历史。

### 正文对象异步清理

归属模块：`content`、`storage`

当前决策：软删除文章或未来覆盖正文时，不立即删除 OSS 对象。

暂缓原因：第一版优先保证数据安全和可恢复，避免误删对象。

未来触发条件：

- OSS 存储成本需要控制。
- 正文版本历史策略明确。
- 增加对象回收任务和保留周期配置。

### 内容审核流

归属模块：`content`

当前决策：第一版状态只包含 `DRAFT`、`PUBLISHED`、`DELETED`，不加入 `REVIEWING`、`REJECTED`。

暂缓原因：审核规则、审核后台和违规处理流程尚未设计。

未来触发条件：

- 平台开放公共发布，需要内容审核。
- 需要人工审核、机器审核或举报处理。

### 发布事件消费

归属模块：`content`、`config`、相关业务模块

当前决策：发布成功时预留事件点，第一版可以先不接 Kafka 消费。

暂缓原因：事件消费者对应的缓存、Feed、搜索、推荐等模块尚未实现。

未来触发条件：

- 任一后续模块需要订阅文章发布、取消发布或删除事件。
